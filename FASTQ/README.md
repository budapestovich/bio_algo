## Обработка FASTQ

В работе анализируется файл формата **FASTQ**, в котором результаты секвенирования ДНК  
Каждое прочтение - **четырьмя строками**:

1. **Идентификатор прочтения**
2. **Нуклеотидная последовательность(sequence)**
3. **Разделитель `+`**
4. **Строка качеств(quality)**, где каждый символ кодирует надежность определения соответствующего нуклеотида

Качество кодируется по шкале **Phred** и переводится в числовое значение по формуле:
**Q = ord(char) - 33**

Чем больше значение **Phred**, тем выше вероятность того, что нуклеотид определён правильно

## Вычисляемые характеристики

Вычисляются:
- общее число прочтений
- минимальная, средняя и максимальная длина прочтений
- **GC-состав** — доля нуклеотидов `G` и `C` в последовательностях
- среднее значение качества **Phred** для заданной позиции по всем прочтениям

## Тримминг по качеству (Скользящее окно)

Параметры:
- ширина окна: **5 нуклеотидов**
- порог среднего качества: **30**

Работа алгоритма:

1. Окно из 5 символов последовательно перемещается вдоль строки качеств
2. Для каждого окна вычисляется **среднее значение Phred**
3. Если среднее качество окна **меньше 30**, то прочтение обрезается до начала этого окна
4. Если плохого окна не найдено, прочтение остаётся без изменений

После тримминга определяется:
- сколько прочтений было обрезано
- минимальная, средняя и максимальная длина полученных последовательностей

## Фильтрация по длине

После тримминга выполняется дополнительная фильтрация:
- прочтения короче **60 нуклеотидов** удаляются.

В результате определяется **окончательное количество прочтений**, прошедших все этапы обработки.
